# Contributing

OpenFugu is currently in the design and architecture stage. The most useful
contributions are concrete use cases, routing policies, evaluation workloads,
trace format proposals, and implementation feedback.

## Good Early Contributions

- Describe a real workload that would benefit from transparent multi-model
  orchestration.
- Propose worker capability fields that are needed for routing decisions.
- Propose policy modes for privacy, cost, latency, quality, or reliability.
- Propose trace fields that make routing decisions easier to inspect.
- Propose evaluation cases where orchestration should beat a fixed model.
- Point out unclear boundaries in the current design documents.

## Contribution Rules

- Keep the project centered on transparent orchestration, not generic chat UX.
- Prefer testable claims over broad product slogans.
- Separate product ideas from implementation commitments.
- Mark speculative ideas clearly.
- Avoid adding production code before the interface and trace model are stable.
